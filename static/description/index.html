AUTO-INFO : Sale Contract Workflow

Contract master, PO reference, and close/reactivate approval flow for Odoo 15

โมดูลนี้เพิ่ม Contract แบบศูนย์กลางให้ทีมขายใช้งานร่วมกับ Sales Order, แยก workflow การปิดสัญญาออกจากปุ่มเดิมบนเอกสารขาย, และบังคับให้ผ่านกระบวนการอนุมัติด้วย Tier Validation ก่อนเปลี่ยนสถานะจริง

จุดเด่นของโมดูล

  • รองรับ 1 Contract ต่อหลาย Customer PO และหลาย Quotation / Sales Order
  • มีสถานะครบ: No Agreement, Active, Close Pending, Closed, Reopen Pending
  • เปลี่ยนจากการปิดสัญญาตรงเป็น Request Close / Request Re-Activate
  • รองรับ Reject -> Acknowledge -> Request ใหม่ ตาม workflow ที่ควบคุมได้
  • บันทึก snapshot ของยอดสัญญา, ยอด invoice, และยอดรับเงินจริงตอนส่งคำขอ
  • จำกัดสิทธิ์ผ่านกลุ่ม Contract Request User เพื่อลดความเสี่ยงการใช้งานผิดคน

โครงสร้างหลัก

  • Contract Agreement: สัญญาแม่ของงานขาย
  • Contract Status Request: เอกสารคำขอเปลี่ยนสถานะสัญญา
  • Customer PO Document: เอกสารอ้างอิง PO ที่ผูกกับ Contract และ SO โดยไม่ซ้ำไฟล์
  • Sale Order Bridge: ปุ่มบน SO สำหรับสร้างหรือเปิดคำขอที่เกี่ยวข้อง
